## roadmap

### 0.1

* api to manage auto scale configuration
* support tsuru metrics backend
* single action expression: "{metric} {operator} {value}"
* scale up / scale down
* integration by bind / unbind
* tsuru plugin

### 0.2

* http backend
* more than one scale (up / down) action
* auto scale disable action
